## Prerequisites

- Java 8 or later
- Maven 3.6+ (for building with the updated plugins)

## Building

```bash
# Compile the project
mvn compile

# Build the JAR with dependencies
mvn package

# Build and install to local Maven repository
mvn install
```

## Testing

```bash
# Run all tests
mvn test

# Run a specific test class
mvn test -Dtest=JsonPersisterTest

# Run tests with verbose output
mvn test -X
```

## Code Quality

Before submitting a pull request, ensure all quality checks pass:

```bash
# Run checkstyle - MUST PASS for merge
mvn checkstyle:check

# Run coverage checks (cobertura)
mvn cobertura:check
```

**Note:** `mvn checkstyle:check` must pass without introducing new violations.
Existing violations in other parts of the codebase are being addressed separately.

## Git Commit Guidelines

This project uses **Conventional Commits** for commit messages. Please follow these conventions:

### Commit Message Format

```
<type>(<scope>): <subject>

<body>

<footer>
```

### Types

- `feat`: A new feature
- `fix`: A bug fix
- `docs`: Documentation only changes
- `style`: Changes that do not affect the meaning of the code (white-space, formatting, missing semi-colons, etc)
- `refactor`: A code change that neither fixes a bug nor adds a feature
- `perf`: A code change that improves performance
- `test`: Adding missing tests or correcting existing tests
- `build`: Changes that affect the build system or external dependencies
- `ci`: Changes to CI configuration files and scripts
- `chore`: Other changes that don't modify src or test files
- `revert`: Reverts a previous commit

### Scope

The scope should be the component affected by the change:
- `hashers` for hashing-related changes
- `persistence` for JSON/persistence changes
- `deps` for dependency updates
- No scope is also acceptable

### Examples

```bash
# Update dependencies
git commit -m "build(deps): update Jackson from 1.9.13 to 2.15.2"

# Fix a bug in the hasher
git commit -m "fix(hashers): handle null File objects in DirHasher"

# Add a new feature
git commit -m "feat: add SHA-3 hashing algorithm support"

# Update Maven plugins
git commit -m "chore: update Maven plugins to latest versions"

# Documentation update
git commit -m "docs: update README with new usage examples"
```

### Co-authored Commits

If using Mistral Vibe or other AI assistants, include the co-authorship footer:

```
Generated by Mistral Vibe.
Co-Authored-By: Mistral Vibe <vibe@mistral.ai>
```

## Pull Requests

When submitting a pull request:

1. Ensure all tests pass (`mvn test`)
2. Ensure checkstyle passes (`mvn checkstyle:check`)
3. Use conventional commit messages
4. Provide a clear description of the changes
5. Reference any related issues
